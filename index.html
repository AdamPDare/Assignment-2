Inline CSS

Inline CSS implementation is where the CSS code is applied to only a single HTML element. This CSS code must be added in the same line as typing the HTML tag and you must use the style element to apply the "style" you want to that specific element. Because of this, it makes Inline CSS less flexible as if you want to change the design of the element you have to edit them individually. An example of inline CSS is shown in the code snippet below:

<h3>Code Extract</h3>
                <h2 style="color:red"></h2>

Internal CSS

Internal CSS implementation is where the CSS code is found within the header section of the HTML page and within the style element. This CSS code will only affect the design of the webpage they are implemented into; this method of implementing CSS allows you to use classes and IDs. This enables you to style and design multiple HTML elements, on the webpage, without repeating lines of code as frequently. Also, using this method enables you to change the design of all the elements that are tagged on the webpage. An example of Internal CSS is shown below and it is the class used to format the code snippets on my webpage:

<h3>Code Extract</h3>
                    <head>
                        <style>
                            code{
                                font-family: monospace;
                                color: chocolate;
                                            
                            } 
                        </style>
                    </head>

External CSS

External CSS implementation is where you use a seperate Cascading Style Sheet (CSS) file; this will allow you to style your website and implement your CSS rules accross numerous HTML files and webpages. This is allows you to make changes to the CSS and it will implement them across the every HTML file that is linked to the CSS file, saving you time and effort. I used External CSS to style my Header, Navigation bar and my columns so I can easily make changes to it and it is implemented across each webpage that I have created. In the code snippet below it shows how I linked the CSS file to enable me to do this:

<h3>Code Extract</h3>
                        <head>
                            <link href="layoutStyleSheet.css" rel"stylesheet">
                        </head>

Assessment of CSS styles

Inline CSS

Inline CSS is a easy and quick method of implementing CSS and CSS rules that affect one html element. This is done via adding a style tag to the html tag; selectors mustnt be used. This method of implementing CSS allows you to easily test or preview any changes you you want to make to your website; also enables you to fix any errors or bugs quickly if needed. Furthermore, this can be done without the need for any external documents needing to be uploaded, unlike external css.

However, this method of implementing CSS should not be used throughout the html file as it will only set rules for one html element, so you would have to address each individual element, which is time consuming; makes the structure of the document look unorganised and messy. Also, individually styling the elements will increase the webpages file size and increase the download times. Lastly, the inline css will only apply to the element upon that specific page, so if you used inline css for the navigation bar, only on that specific page the rules will be applied to the navigation bar on one webpage.

So, Inline CSS should only be used to test and preview any changes you want to make to your document; quick-fix any errors or bugs that could be ocurring.

Internal CSS

Internal CSS is an easy way of grouping various HTML elements via the use of ID selectors, and enable you to apply CSS rules to all the grouped elements. Inline CSS is implemented within the <head> section with the use of the <style> tag. This then allows you to add all the CSS code needed into just one HTML file, meaning you will not have to upload numerous files to display your webpage.

The only issue with Internal CSS implementation is that, since you have added the code directly to the HTML file, it will increase the webpages size and loading times. Furthermore, the number of lines will substantially increase, in the html file, making it harder to find specific parts of the code as the two coding languages are not seperated.

In my opinion, you should only use Internal CSS if you want to apply CSS rules to multiple HTML elements, that are specific to that webpage and not universally used across the website, for example: the navigation bar should be created using external CSS.
